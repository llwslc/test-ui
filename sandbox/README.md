# Sandbox —— 主题世界演示与图集

> 一条视觉方向做成一个点开就能玩的单文件网页，再用一个图集把它们全部串起来。产出落在以你名字命名的目录 `sandbox/<agent>/`，与兄弟目录平级，各自从零写。

## 1 交付清单

- `sandbox/<agent>/<theme-id>.html` —— 每条主题一个，扁平命名，无子目录。
- `sandbox/<agent>/index.html` —— 图集，串起同目录全部主题。

## 2 主题来源

`prompt/theme/README.md` 是唯一来源，目录里每一条都做，一条不落。每条按「配色；字体；形状与布局；纹理与效果；动效」五面写就，五面全部兑现到像素：点名的颜色、字体气质、形状语汇、纹理手法、动效动词，逐项在页面上找得到对应。

`theme-id` 取条目代号小写，`PRISM` 落成 `prism.html`；条目上的链接文件名只是 kit 文档去向，不参与命名。「已建」与「候选」只标记 kit 落地状态，与本目录无关，同等对待。

## 3 单套主题页

### 3.1 技术形态

单文件自包含：HTML + 内联 `<style>` + 内联 `<script>` + 内联 SVG，双击即开，零构建、零打包、零框架。外部资源只留 Google Fonts 的 `<link>`；图标、纹理、装置一律内联 SVG 或纯 CSS 画出来。交互全部原生 JS。

### 3.2 页面骨架

顶部一条世界内的状态栏或铭牌；Hero 给主标题、副文、一到两枚主行动钮、一件招牌装置；控件展区分块铺开全部控件，每块配一个世界内的小标题；页脚一句收尾。

### 3.3 控件清单

每套齐活：按钮五变体含 disabled、文本输入、单选组、复选、开关、滑杆、自绘下拉弹层、标签页、进度条、量表、模态、toast、tooltip。每件都真能动——值会变、状态留得住、弹层开得了也关得掉、下拉键盘可选、模态锁得住焦点。

### 3.4 控件设计法则

每个控件是那个世界里真会被拨动的一件实物：开关做成眼睛开阖、阀门闸把、机械椎骨，量表做成压力表、应援槽、食相盘。一件控件一副独立造型，同一个母题不重复塞进圆圈、方框、胶囊凑数。形态自证身份，遮住标签也认得出这是开关还是单选。rest、hover、focus、active、disabled、checked 六态肉眼可分。全页至少留一记招牌一击：只属于这套主题、看一眼就记住的交互或画面。

### 3.5 世界观

产品名、栏目名、按钮文案、提示语、占位符全部长在主题里，通篇同一个声音，读起来像那个世界真发的产品页。

## 4 图集 index.html

### 4.1 首页

卡片网格，每卡一套主题：实时缩略图 + 编号 + 代号 + 中英文名。缩略图用活页面而非截图——`<iframe>` 按 1280 逻辑宽加载，`transform: scale()` 按「容器宽 / 1280」缩放，容器 `overflow: hidden` 只裁出 hero 那一屏，`pointer-events: none` 让整张卡可点。`IntersectionObserver` 进视口才开始加载，并发上限 4，排队泵出，加载完落一个状态类。

### 4.2 详情

点卡进入，铺满窗口：左侧主题目录，右侧顶栏加预览区。目录只用文字——编号、代号、中文名，当前项高亮。顶栏依次是当前主题标识、左右切换、PC／Mobile 视口、全屏、Open 新窗打开。PC 视口下 iframe 铺满预览区，Mobile 视口下 390×844 圆角设备框居中，选择记进 `localStorage` 下次沿用。每次换主题预览区先盖一层加载态，iframe `load` 后揭开。

### 4.3 网页级全屏

「全屏」按钮排在 Open 之前。按下后主题 iframe 铺满整个 index，侧栏与顶栏一并隐藏；左右方向键照常切主题并保持全屏；`Esc` 退出全屏回到详情，再按一次回首页。全屏态留一组淡色浮层：右上退出按钮、左右两侧切换箭头，hover 提亮。这是页面内铺满，不调用浏览器全屏 API。

### 4.4 导航

`#theme-id` 深链直达该主题详情；← → 全程切换主题；窄屏 760 以下侧栏转抽屉，汉堡钮唤出、蒙层点掉。

## 5 质量底线

- 桌面 1440 与手机 390 两个宽度都零横向溢出，手机宽度单列铺开。
- console 零 error。
- 键盘可达：Tab 走遍所有控件，焦点框清晰可见，弹层 `Esc` 关得掉。
- `prefers-reduced-motion` 下动效降为瞬时或极弱。
- 用户可见文本一律 `textContent` 注入。
- 正文与标签对比度 4.5:1 起；以响度立意的主题，签名色保住饱和。
- 无 JS 时内容照样呈现。

## 6 自检与交付

每套做完驱动真实浏览器自检一遍：两个宽度各看一眼、点开每个弹层、切过每个状态、收齐 console。临时截图与调试脚本不入库，只提交 `.html`。全部完成后在图集里通拉一遍，确认缩略图逐张加载、逐套点进去无碍。
