# Sandbox —— 主题世界演示与图集

> `sandbox/<agent>/` 是一个主题演示目录：每套主题做成一个单文件网页，双击就能打开试玩，再由一个图集把它们串起来。本规范描述目录结构与每类文件的做法。做哪套主题听指派，新增一条方向就补一套并登记进图集。

## 1 目录结构

```
sandbox/
├── README.md
├── claude/
│   ├── index.html
│   ├── nova.html
│   ├── abyss.html
│   └── ...
└── codex/
    ├── index.html
    └── ...
```

- `README.md` —— 本规范，全体 agent 共用。
- `<agent>/` —— 一个 agent 一个目录，以自己的名字命名，各自从零写，互不参照。
- `<agent>/index.html` —— 图集，一个目录一份：主题注册表、首页网格、详情预览。
- `<agent>/<theme-id>.html` —— 一套主题一个文件，文件名即 `theme-id`，扁平摆放，无子目录。
- 主题页各自自包含：不共享 CSS 与 JS 文件，不引用兄弟主题，不留构建产物。

## 2 主题来源与选题

`prompt/theme/README.md` 是唯一主题来源。每条按「配色；字体；形状与布局；纹理与效果；动效」五面写就，五面全部兑现到像素：点名的颜色、字体气质、形状语汇、纹理手法、动效动词，逐项在页面上找得到对应。条目最大——它点名的做法与本规范的通用建议冲突时，以条目为准。

`theme-id` 取条目代号小写，`PRISM` 落成 `prism.html`；条目上的链接文件名只是 kit 文档去向，不参与命名。「已建」与「候选」只标记 kit 落地状态，与本目录无关，同等对待。

做哪套听指派：

- 指名某套 —— 只做那一套；目录里已有同名文件就整套重做，不在旧稿上打补丁。
- 目录新增一条方向 —— 只补新增的那套，其余原样不动。
- 明确要求全量 —— 逐条通做。

## 3 新增或重做一套主题

1. 读 `prompt/theme/README.md` 里那一条，把五面拆成落地清单。
2. 照 §4 写出 `<theme-id>.html`。
3. 照 §5.1 在图集注册表追加或更新该套的一行。
4. 照 §7 自检，提交主题页与注册表那一行改动。

## 4 单套主题页

### 4.1 技术形态

单文件自包含：结构、样式、脚本、SVG 全写在同一个 `.html` 里，双击即开，零构建、零打包、零框架。外部资源只留 Google Fonts 的 `<link>`；图标、纹理、装置一律内联 SVG 或纯 CSS 画出来。交互全部原生 JS。

### 4.2 版心与留白

正文与控件排在一条居中版心里，最宽 1180px。窗口再宽，只是两侧留白变多，版心不跟着拉长。

任何窗口宽度下文字都不贴边：桌面两侧各留 24px 起，手机各留 16px 起。铺满屏宽的状态栏、页脚、跑马灯，里头的文字同样按这个留白排。

### 4.3 页面骨架

顶部一条世界内的状态栏或铭牌；Hero 给主标题、副文、一到两枚主行动钮，外加一处招牌画面；控件展区分块铺开全部控件，每块配一个世界内的小标题；页脚一句收尾。

### 4.4 控件清单

每套齐活：按钮五变体含 disabled、文本输入、单选组、复选、开关、滑杆、自绘下拉弹层、标签页、进度条、量表、模态、toast、tooltip。每件都真能动——值会变、状态留得住、弹层开得了也关得掉、下拉键盘可选、模态锁得住焦点。

### 4.5 控件设计法则

控件造型从这套主题自己的语汇里长出来，不是通用控件换个配色。按主题性格走两条路：具象的主题把控件做成那个世界里真会被拨动的实物，开关是眼睛开阖、阀门闸把、机械椎骨，量表是压力表、应援槽、食相盘；形式的主题从自己的几何、网格、印刷或笔法体系里推出形态，包豪斯用方与圆，位图用像素格，报章用栏线与字号——克制本身就是这类主题的招牌，不往上硬塞道具。

两条路共同守住：一件控件一副独立造型，同一个母题不重复塞进圆圈、方框、胶囊凑数；形态自证身份，遮住标签也认得出这是开关还是单选；rest、hover、focus、active、disabled、checked 六态肉眼可分。

全页至少留一记招牌一击：只属于这套主题、看一眼就记住的交互或画面。具象的主题多半是一件会动的装置，形式的主题可以是一套贯穿全页的构成或排印规则。

### 4.6 世界观

产品名、栏目名、按钮文案、提示语、占位符全部长在主题里，通篇同一个声音，读起来像那个世界真发的产品页。

## 5 图集 index.html

图集建一次即长期沿用，此后新增主题只动 §5.1 的注册表。

### 5.1 主题注册表

图集顶部维护一份数组，一套主题一行 `{id, code, name}`：`id` 与主题页文件名一致，`code` 是代号，`name` 取条目的中英文名。编号、首页卡片、侧栏目录、左右切换、深链全部由这份数组按顺序派生，别处不再出现主题名单。新增主题追加一行即全站生效。

### 5.2 首页

卡片网格，每卡一套主题：上半是缩略图，下半标出编号、代号与中英文名。

缩略图是那套主题真在运行的页面，不是截图，动效照跑。它按桌面版式渲染再等比缩进卡片，只露出 hero 那一屏；鼠标落在缩略图上点到的仍是整张卡。卡片滚进视口才开始加载，同时最多加载四五套，没加载完的卡片先显示占位。

### 5.3 详情

点卡进入，铺满窗口：左侧是主题侧栏，右侧上方顶栏、下方预览区。

侧栏从上到下三段。顶端一枚返回键写明「全部主题」，点它回首页。其下一个搜索框，输入即时过滤，代号、中英文名、`theme-id` 任一命中就留下，一条都不中时列表原位显示一行空态。再往下是主题目录，只用文字——编号、代号、中文名，当前项高亮，编号跟着注册表顺序走，过滤时不重排。搜索框按 `/` 聚焦，回车进入第一条命中，`Esc` 先清空再离焦；焦点在输入框里时左右方向键归输入框，不切主题。

顶栏依次是当前主题标识、左右切换、PC／Mobile 视口、全屏、Open 新窗打开。PC 视口下预览铺满预览区；Mobile 视口固定 390 宽，装进圆角设备框居中，高度取可用空间，最高 844。视口选过一次，下次进来照旧。换主题时预览区先显示加载态，页面就绪再揭开。

### 5.4 网页级全屏

「全屏」按钮排在 Open 之前。按下后主题铺满整个 index，侧栏与顶栏一并隐藏，视口沿用当前选择。左右方向键照常切主题并保持全屏；`Esc` 退出全屏回到详情，再按一次回首页。全屏态留一组淡色浮层：右上退出按钮、左右两侧切换箭头，hover 提亮。铺满的是 index 这一页，不是浏览器客户端全屏。

### 5.5 导航

`#theme-id` 深链直达该主题详情；← → 全程切换主题；窄屏 760 以下侧栏转抽屉，汉堡钮唤出、蒙层点掉。`Esc` 逐级后退：搜索框内先清空、再离焦，然后退出全屏，再关抽屉，最后回首页。

## 6 质量底线

- 桌面 1440 与手机 390 两个宽度都零横向溢出，手机宽度单列铺开。
- 两个宽度下都没有文字贴住视口边缘，横向留白按 §4.2 的下限。
- console 零 error。
- 键盘可达：Tab 走遍所有控件，焦点框清晰可见，弹层 `Esc` 关得掉。
- `prefers-reduced-motion` 下动效降为瞬时或极弱。
- 用户可见文本一律 `textContent` 注入。
- 正文与标签对比度 4.5:1 起；以响度立意的主题，签名色保住饱和。
- 无 JS 时内容照样呈现。

## 7 自检与交付

每套做完驱动真实浏览器自检一遍：两个宽度各把整页滚完、动效走完再判断，点开每个弹层，切过每个状态，收齐 console。临时截图与调试脚本不入库，只提交 `.html`。动过图集的，回首页确认新卡缩略图加载、点进去左右切换与深链都通。
